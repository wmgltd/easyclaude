Multi-session terminal hub for Claude Code.

Why

Claude Code is great. Running six of them in tmux windows is not.

PikudClaude puts every Claude Code session in one window. Sidebar to navigate, ⌘+number to switch, drag to reorder. tmux keeps them alive in the background so sessions survive app restarts.

Features

Built for people who keep a lot of Claudes spinning.

⌘K

Command palette

Fuzzy palette for every action. Session-only switcher on ⌘P. ⌘1–⌘9 jumps directly.

🔔

Awaiting alerts

Detects when Claude asks a numbered-options question and chimes / fires a macOS notification / bumps the dock badge — only when the session isn't focused.

♻️

Survives restarts

tmux keeps sessions running in the background. PikudClaude re-attaches on launch and forces a clean redraw so Claude's TUI doesn't double-render.

🔖

Bookmarks & search

Bookmark any point in a session with ⌘B. Search inside scrollback with ⌘F. Auto-bookmark when Claude pauses awaiting input.

🔗

Click to open

Click any path/to/file.ts:42 in the terminal output to open it in Cursor or VSCode at the exact line.

🎨

Themes & fonts

Default, Solarized Dark, Dracula, Nord, Light, or fully custom. Font picker with Hebrew-friendly options. Quiet hours for notifications.

The differentiator

Hebrew and Arabic, actually working.

xterm.js issue #701 ("Support RTL languages") has been open since 2017. Every Electron terminal that uses xterm.js inherits the gap — VSCode, Cursor, Hyper, Tabby, Wave, Mux. iTerm2 and WezTerm have their own RTL bugs. Only Terminal.app handles bidi properly, and it has no multi-session UX.

PikudClaude solves it via the DOM renderer + a MutationObserver on xterm rows + targeted direction: rtl; unicode-bidi: isolate. Not a full UAX#9 implementation, but enough that you can read Claude's Hebrew responses and type Hebrew into the prompt without giving up on your first language.

Stack

Boring, on purpose.

  • Electron 33 + Vite (electron-vite)
  • React 18 + TypeScript
  • xterm.js 5.5 DOM renderer + fit/search/clipboard/web-links addons
  • node-pty for the actual PTY
  • tmux (system-installed) as the session-persistence layer
